Detailed Description

Through the streaming interface, Sphero provides real-time location data in the form of (X,Y) coordinates on the ground plane. When Sphero wakes up it has coordinates (0,0) and heading 0, which corresponds to facing down the positive Y-axis. This command allows you to move Sphero to a new location and change the alignment of locator coordinates with IMU headings.

When Sphero receives a Set Heading command it changes which real-world direction corresponds to heading 0. By default, the locator compensates for this by modifying its value for yaw tare. For instance, if you wake up Sphero and drive straight, you will be driving down the Y-axis. If you use the Set Heading feature in the drive app to turn 90 degrees, you will still have heading 0, but the locator knows you have turned 90 degrees and are now facing down the X-axis. This feature can be turned off, in which case the locator knows nothing about the Set Heading command. This can lead to some strange results. For instance, if you drive using only roll commands with heading 0 and set heading commands to change direction the locator will perceive your entire path as lying on the Y-axis.

Initializer to set the parameters used to configure the locator.

Parameters
flagDetermines whether calibrate commands automatically correct the yaw tare value. 0x00 for NO, 0x01 for YES?
newX(in cm) The new x value that the device will equate the current x value to.
newY(in cm) The new y value that the device will equate the current y value to.
newYaw(in degrees) The value will be interpreted in the range 0-359 inclusive. Controls how the X,Y-plane is aligned with Sphero’s heading coordinate system. When this parameter is set to zero, it means that having yaw = 0 corresponds to facing down the X- axis in the positive direction.
